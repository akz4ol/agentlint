## Responsibilities

Maintainers are responsible for:

- Reviewing and merging pull requests
- Triaging issues
- Releasing new versions
- Maintaining documentation
- Enforcing the Code of Conduct

## Becoming a Maintainer

We welcome new maintainers! The path:

1. **Contributor**: Make quality contributions
2. **Regular Contributor**: Consistent, high-quality PRs over time
3. **Maintainer**: Invited based on sustained contribution

## Decision Making

- **Minor changes**: Any maintainer can merge
- **Major changes**: Require discussion and consensus
- **Breaking changes**: Require RFC and review period
